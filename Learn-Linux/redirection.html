1. 重定向

1.1. I/O 重定向

  • 可以把命令行的输入重定向为从文件中获取内容

  • 可以把命令行的输出结果重定向到文件中

  • I/O 重定向可以改变输出内容发送的目的地，也可以改变输入内容的来源地

    • 通常来说，输出内容显示在屏幕上，输入内容来自于键盘

1.2. 标准输入、标准输出和标准错误

  • 标准输出(standard output)

    • stdout

  • 标准错误(standard error)

    • stderr

  • 标准输入(standard input)

    • stdin

  • 默认情况下，标准输出和标准错误都被输出到屏幕上，不会保存再磁盘中

  • 默认情况下，标准输入连接到键盘

1.3. 标准输出重定向

  • 处理不想要的输出

    • 通过把不想要的输出重定向到特殊文件 /dev/null 来实现

  • 注意

    • 重定向时，只有 command、file 左右可以加空格
    • 注意这些格式：2>、2>&1、&> file

>

  • 重定向标准输出(stdout)内容的目的地

  • 每次重定向，目的文件都会被新内容重新覆盖

command > file

1.3.1. >>

  • 从文件的尾部开始添加标准输出内容

  • 如果目的文件不存在，会创建目的文件

command >> file

1.4. 标准错误重定向

  • 一个程序可以把生成的输出内容发送到任意文件流中

  • shell 使用文件描述符索引文件流，前 3 个分别为

    • 0 标准输入文件
    • 1 标准输出文件
    • 2 标准错误文件

  • shell 提供了使用文件描述符编号来表示重定向文件的方法

  • 2> 和 2>>

    • 注意没有空格

1.4.1. 将标准输出和标准错误重定向到同一个文件

  • 方法一
command > file 2>&1
# 首先重定向标准输出到 file 中
# 然后使用 2>&1 把标准错误重定向到标准输出中
# 执行两个重定向，执行效率低，重定向顺序不能换
  • 方法二
command &> file
# &> 会同时把标准输出和标准错误都重定向到 file 中
# 效率比方法一更高

1.5. 标准输入重定向

  • 使用 <
command < file # 重定向标准输入(stdin)的源
  • 示例
cat < file

1.6. 管道

  • 使用 |
# 把一个命令的标准输出传送到另一个命令的标准输入中
command1 | command2
  • 示例
ls -l /usr/bin | less

1.7. 过滤器

  • 过滤器接受输入，按照某种方式对输入进行改变，然后再输出它

  • 相关过滤器命令(参考)

    • sort：排序
    • uniq： 报告或忽略文件中重复的行
