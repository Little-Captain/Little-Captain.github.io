1. 扩展

  • 什么是扩展？
    • 如，执行某个命令前，shell 会对该命令中的通配符(*)进行扩展

1.1. 路径名扩展

  • 通过使用通配符来实现扩展的机制称为路径名扩展

  • 示例

echo *
echo D*
echo *s
echo [[:upper:]]*
echo /usr/*/share

1.2. 波浪线(~)扩展

  • 被扩展为指定用户的主目录名

    • ~username

  • 如果没有指定用户名，则扩展为当前用户的主目录

1.3. 算术扩展

  • shell 支持通过扩展来运行算术表达式

    • 算术扩展只支持整数
    • 空格在算术表达式中没有意义
    • 算术表达式可以嵌套

  • 一些支持的操作符

# 扩展格式
$((expression))
  • 示例
echo $((2 + 2))
echo $(((5 ** 2) * 3))
echo $((5 / 2))

1.4. 花括号扩展

  • 花括号扩展可以按照花括号内的模式创建多种文本字符串

  • 花括号表达式可以包含一系列逗号分隔的字符串，也可以包含一系列整数或单个字符

  • 花括号扩展支持嵌套

  • 格式

{str1,str2,str3...} # 内部字符串中有空格字符需要进行转义
{1..5} # .. 表示区间
{5..1} # 可倒序
  • 示例
echo Front-{A,B,C}-Back
echo Number_{1..5}
echo {Z..A}
echo a{A{1,2},B{3,4}}b
echo {A..Z}{0..10}
mkdir {2009..2011}-0{1..9} {2009..2011}-{10..12}

1.5. 参数扩展

  • 格式
# 对于参数扩展，如果变量名拼写错误，不会报错，只不过是输出一个空字符串而已
$parameter
  • 示例
echo $USER

1.6. 命令替换

  • 命令替换可以把一个命令的输出作为一个扩展模式使用

  • 格式

$(command) # 推荐语法
`command` # 早期语法
  • 示例
echo $(ls)
ls -l $(which cp)
file $(ls /usr/bin/* | grep zip)
ls -l `which cp`

1.7. 引用

  • shell 提供了一种称为引用(quoting)的机制，用来有选择性地避免不想要的扩展

1.7.1. 双引号

  • 把文本放在双引号中

    • shell 中使用的大部分特殊字符都将失去它们的特殊含义
    • 保留特殊含义的字符有：$ \ `

  • 双引号中，单词分隔、路径名扩展、波浪线扩展和花括号扩展都将失效

  • 单词分隔

    • 在 shell 中，空白字符会把一个字符串分割为多个单词
    • 使用双引号可以阻止单词分隔
    • 默认情况下，单词分隔会先查找是否存在空格、制表符以及换行符，然后把它们当作单词间的界定符(delimiter)。
      • 也就是没有用双引号包含起来的空格、制表符和换行符不会被当作文本的一部分，而只是被当成分隔符。
      • 如果加上双引号，单词分隔功能将失效，嵌入的空白字符将不再被当成界定符，而是被当成参数的一部分
echo $(cal)
echo "$(cal)"

  • 参数扩展、算术扩展和命令替换有效

  • 使用双引号可以处理文件名中包含空白的情况

  • 示例

echo "$USER $((2 + 2)) $(cal)"

1.7.2. 单引号

  • 使用单引号，可以抑制所有的扩展

  • 示例

echo text /mnt/d/Code/js/base64/*.json {a,b} $(echo foo) $((2 + 2)) $USER
echo "text /mnt/d/Code/js/base64/*.json {a,b} $(echo foo) $((2 + 2)) $USER"
echo 'text /mnt/d/Code/js/base64/*.json {a,b} $(echo foo) $((2 + 2)) $USER'

1.8. 扩展中的转义字符

  • 以 \ 开头来定义一个转义字符

  • 转义字符经常在双引号中用来有选择性地阻止扩展

  • 转义字符也常用来消除文件名中某个字符的特殊含义

  • 单引号中的反斜杠没有特殊含义，就是一个普通的字符
