1. 权限相关命令

1.1. umask

  • 设置默认权限

  • umask 命令控制着创建文件/文件夹时指定给文件/文件夹的默认权限

    • 使用八进制表示法来表示从文件/文件夹模式属性中需要减去的值

  • 使用 4 个八进制数字表示

  • 四个数字分别对应

    • 特殊权限
    • user
    • group
    • others

  • 特殊权限

    • 第 1 位：setuid
    • 第 2 位：setgid
    • 第 3 位：sticky
    • 设置特殊权限
      • 设置特殊权限一般通过符号表示法，因为这样可以很方便的做到不影响一般权限
      • 授予程序 setuid 权限：chmod u+s 程序名
        • 查看结果：-rwsr-xr-x
      • 授予程序 setgid 权限：chmod g+s 目录
        • 查看结果：drwxrwsr-x
      • 授予程序 sticky 权限：chmod +t 目录
        • 查看结果：drwxrwxrwt

  • 示例

    • 0002
      • 创建文件：666 - 002 = 664(rw-rw-r--)
      • 创建文件夹：777-002 = 775(rwxrwxr-x)
    • 0027
      • 创建文件：666 - 027 = 640(rw-r-----)
      • 创建文件夹：777-027 = 750(rwxr-x---)

1.2. chmod

change file mode bits

  • 用于更改文件或目录的模式(权限)

    • 只有所有者和超级用户才可以更改文件或者目录的模式

  • 使用 -R 选项对文件夹进行递归改变

  • 支持两种不同的更改文件模式的方式

1.2.1. 八进制数字表示法

  • 使用八进制数字来设置所期望的权限模式

  • 通过使用 3 位八进制数字，可以分别设置所有者、组成员和其他所有用户的权限
    • 严格的说是 4 位，只是第 1 位是特殊权限，一般都省略了
    • 4 位分别对应
      • 特殊权限
      • user
      • group
      • others
chmod 664 README.md

1.2.2. 符号表示法

  • 该表示法由三部分组成
    • 要设置谁的权限。如果没有指定，则假定使用 all
    • 要执行哪个操作
      • +：添加权限
      • -：删除权限
      • =：用指定权限覆盖原有权限
    • 要设置哪种权限
      • r：读
      • w：写
      • x：执行

chmod u+x push.sh
  • 示例

  • 优点：允许设置单个属性，而不影响其他属性

1.3. chown

change file owner and group

  • 更改文件所有者和所属群组

  • 使用 chown 需要超级用户的权限

  • 使用 -R 选项对文件夹进行递归改变

  • 使用格式

chown [owner][:[group]] file...
  • 注意理解冒号的作用
    • bob 不改变群组
    • bob: 同时改变群组为 bob 登录系统时所属的组

1.4. chgrp

change group ownership

  • 更改文件所属群组

  • 使用 -R 选项对文件夹进行递归改变

  • 早期 UNIX 中，chown 只能更改文件/文件夹的所有者，不能改变其所属群组，这时可以使用 chgrp 来修改其所属群组
