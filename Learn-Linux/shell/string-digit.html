1. 字符串和数字

1.1. 参数扩展(Parameter Expansion)

1.1.1. 基本参数

  • 参数扩展的最简单形式体现在平常对变量的使用中。

  • 当变量相邻于其他文本时, 必须使用花括号包裹, 否则可能让 shell 混淆。

  • 示例

a=foo
echo "$a_file" # 
echo "${a}_file" # foo_file

1.1.2. 空变量扩展的管理

  • 有的参数扩展用于处理不存在的变量和空变量。这些参数扩展在处理缺失的位置参数和给参数赋默认值时很有用处。

扩展形式

  • 形式1
${parameter:-word}

  • 如果 parameter 未被设定(如不存在)或者是空参数, 则其扩展为 word 的值; 如果 parameter 非空, 则扩展为 parameter 的值。

  • 形式2

${parameter:=word}

  • 如果 parameter 未被设定或者为空, 则其扩展为 word 的值; 此外, word 的值也将赋给 parameter。如果 parameter 非空, 则扩展为 parameter 的值。

  • 位置参数和其他特殊参数不能以这种方式赋值。

  • 形式3

${parameter:?word}

  • 如果 parameter 未设定或为空, 这样扩展会致使脚本出错而退出, 并且 word 内容输出到标准错误。如果 parameter 非空, 则扩展结果为 parameter 的值。

  • 形式4

${parameter:+word}
  • 若 parameter 未设定或为空, 将不产生任何扩展。若 parameter 非空, word 的值将取代 parameter 的值; 然而, parameter 的值并不发生变化。

1.1.3. 返回变量名的扩展(bash 支持，zsh 不支持)

  • shell 具有返回变量名的功能。这种功能在相当特殊的情况下才会被用到。

  • 扩展形式

${!prefix*} 或 ${!prefix@}
  • 该扩展返回当前以 prefix 开头的变量名。根据 bash 文档, 这两种扩展形式执行效果一模一样。

1.1.4. 字符串操作

  • 对字符串的操作, 存在着大量的扩展集合。其中一些扩展尤其适用于对路径名的操作。

  • 参数扩展是一个比较重要的功能。进行字符串操作的扩展可以替代其他常用的命令。扩展通过取代外部程序, 通常可以改善脚本的执行效率。

扩展1

${#parameter}

  • 扩展为 parameter 内包含的字符串的长度。一般来说, 参数 parameter 是个字符串。然而, 如果参数 parameter 是 "@" 或 "*", 那么扩展结果就是位置参数的个数。

  • 示例

foo="This string is long."
echo "'$foo' is ${#foo} characters long."

扩展2

${parameter:offset}
${parameter:offset:length}

  • 这个扩展用来提取一部分包含在参数 parameter 中的字符串。扩展以 offset 字符开始, 直到字符串末尾, 除非 length 特别指定。

  • 示例

foo="This string is long."
echo ${foo:5} # string is long.
echo ${foo:5:6} # string

  • 如果 offset 的值为负, 默认表示它从字符串末尾开始, 而不是字符串开头。注意, 负值前必须有一个空格, 以防和 "$" {parameter:-word} 扩展混淆。如果有 length 的话, length 不能小于 0。

  • 如果参数是 "@", 扩展的结果则是从 offset 开始, length 为位置参数。

扩展3

${parameter#pattern}
${parameter##pattern}

  • 根据 pattern 定义, 这些扩展去除了包含在 parameter 中的字符串的主要部分。pattern 是一个通配符模式, 类似那些用于路径名的扩展。

  • 两种形式的区别在于 "#" 形式去除最短匹配, 而 "##" 形式去除最长匹配。

  • 示例

foo=file.txt.zip
echo ${foo#*.} # txt.zip
echo ${foo##*.} # zip

扩展4

${parameter%pattern}
${parameter%%pattern}

  • 这些扩展与 "#" 和 "##" 扩展相同, 只是它们是从参数包含的字符串末尾去除文本, 而非字符串开头。

  • 示例

foo=file.txt.zip          
echo ${foo%.*} # file.txt
echo ${foo%%.*} # file

扩展5

${parameter/pattern/string}
${parameter//pattern/string}
${parameter/#pattern/string}
${parameter/%pattern/string}

  • 这个扩展在 parameter 的内容上执行搜索和替换非常有效。

  • 如果文本被发现和通配符 pattern 一致, 就被替换为 string 的内容。

  • 在 "/" 形式下, 只有第一个出现的 pattern 被替换。

  • 在 "//" 形式下, 所有 pattern 都被替换。

  • "/#" 形式要求匹配出现在字符串开头。

  • "/%" 形式要求匹配出现在字符串末尾。

  • "/string" 可以省略, 不过和 pattern 匹配的文本都会被删除。

  • 示例

foo=JPG.JPG
echo ${foo/JPG/jpg} # jpg.JPG
echo ${foo//JPG/jpg} # jpg.jpg
echo ${foo/#JPG/jpg} # jpg.JPG
echo ${foo/%JPG/jpg} # JPG.jpg

1.2. 算术计算和扩展

1.2.1. 算术扩展的基本形式

# 其中 expression 是一个有效的算术表达式。
$((expression))

1.2.2. 数字进制

  • 在算术表达式中, shell 支持任何进制表示的整数。

  • 不同的数字进制

  • 示例
echo $((0xff)) # 255
echo $((2#11111111)) # 255

1.2.3. 一元运算符

  • + 用于指示一个数字是正数。

  • - 用于指示一个数字是负数。

1.2.4. 简单算术运算符

  • shell 的算术运算符仅适用于整数, 所以除法的结果永远是整数。

1.2.5. 赋值

  • 算术表达式可以用来进行赋值。

  • 示例

(( foo = 5))
# 这个算术表达式做了两件事
# (1) 它给变量 foo 赋值 5
# (2) 因为赋值是成功的, 所以退出状态为 0
echo $? # 赋值成功，退出状态码为 0
echo $foo # 5
  • 赋值操作符

1.2.6. 位操作

  • 位操作符被用于特定的低层任务中, 常用于位标志的设置和读取。

  • 除了按位取反之外, 也存在相应的赋值操作(如 <<=)。

  • 示例

echo $((1<<1)) # 2
echo $((1<<2)) # 4

1.2.7. 逻辑操作

  • 当使用逻辑操作时, 表达式遵循算术逻辑的规则。这就是说, 值为零的表达式为 false, 而非零表达式为 true。

1.3. bc：一种任意精度计算语言

  • 单纯使用 shell 无法实现浮点运算。

  • 为了让 shell 支持浮点运算, 需要使用一个外部程序。这里有几种方法可以使用, 如嵌入 Perl 或 AWK。

  • 另一种方法是使用一个专门的计算器程序, 大多数 Linux 系统都支持这种程序 bc。

  • bc 程序读取一个使用类 C 语言编写的程序文件, 并执行它。bc 脚本可以是一个单独的文件, 也可以从标准输入中读取。bc 语言支持很多功能, 包括变量、循环以及由程序员自定义的函数。

1.3.1. 简单使用

  • 直接运行 bc，可以交互地使用 bc。

  • 使用 bc script.bc 执行 bc 脚本。这样输出结果后，会进入交互界面。

bc foo.bc
  • 通过 -q 选项，可以禁止打印版本信息。
bc foo.bc -q
  • 使用 bc < script.bc 执行 bc 脚本。这样只会输出结果，不会进入交互界面。
bc < foo.bc

  • 在交互界面，通过 quit 退出。也可以使用 Ctrl-D。

  • 因为 bc 支持标准输入, 所以也可以使用嵌入文档、嵌入字符串和管道传递脚本。

# 嵌入文档
bc << _EOF_
2 + 2
3 * 2.1
_EOF_
bc <<< "2+2" # 嵌入字符串
echo "2 + 2" | bc # 管道符
  • bc 中的计算精度是由 bc 脚本中 scale 变量的值决定的。scale 用于确定小数点后的位数。scale 默认为 0。scale 只对其改变后的语句有效。
